# 策略分析报告（workflow 阶段一第 5 步）详细说明

对应 `workflow.md` 中的：

- `node processors/strategyAnalyzer.js`
- 输出到 `strategy/` 目录下 4 个分析文件（另含 `README.md` 索引）

实现位置：`cup-analyzer/crawler-server/processors/strategyAnalyzer.js`，继承 `BaseCrawler`。

---

## 1. 在哪执行

`node processors/strategyAnalyzer.js` 是相对于 **`cup-analyzer/crawler-server`** 目录的（与 `crawler-server/README.md` 一致）。需要先进入该目录再执行：

```bash
cd cup-analyzer/crawler-server
node processors/strategyAnalyzer.js
```

环境变量 `CUP_ANALYZER_CUP` 默认是 **`theWorldCup`**（见 `crawler-server/config/index.js`），一般无需额外设置。若曾切换到其他杯赛/联赛，需显式指定：

```bash
CUP_ANALYZER_CUP=theWorldCup node processors/strategyAnalyzer.js
```

---

## 2. 脚本总流程

`StrategyAnalyzer.generateFullReport()` 大致步骤：

1. **`parseScheduleData()`**（定义在 `crawler-server/crawlers/base.js`）：读取当前杯赛对应的赛程 JS。世界杯为 **`theWorldCup/data/c75.js`**，解析出 `arrTeam`、`rounds`（赛程/积分榜等）等结构。
2. 若解析失败 → 打错误日志并结束。
3. 输出目录：**`config.paths.cupAnalyzer/strategy`**。在世界杯配置下即 **`cup-analyzer/theWorldCup/strategy/`**。
4. 依次生成 4 份策略分析 Markdown，并调用 **`generateStrategyReadme`** 写入 **`strategy/README.md`**。

**输入**：主要是赛程数据 **`c75.js`**。  
**静态规则**：脚本内嵌 **2026 世界杯 32 强对阵表** `knockoutBracket`（与 `rule/16th-finals-rule.md` 等文档一致），用于推演小组名次对应的淘汰赛路径。

---

## 3. 四个分析文件分别做什么

| 文件 | 生成函数 | 内容要点 |
|------|----------|----------|
| `group-path-analysis.md` | `generateGroupPathReport` | 对 A～L 各组，按内置对阵表计算 **小组第 1 / 第 2 / 第 3（最佳第三）** 在 32 强可能落入 **上半区/下半区**、对手占位符（如 `3rd_ABCDF`）、场次与时间。**以赛制为主**，不依赖实时积分。 |
| `opponent-selection.md` | `generateOpponentSelectionReport` | 说明「挑对手」逻辑；按组若有积分榜 `S27970{字母}`，用 `arrTeam` 映射中文队名，表格对比若拿 **小组第一 vs 第二** 时 32 强大致对手。**积分榜随赛程数据更新而变化**。 |
| `tacit-match-detection.md` | `generateTacitMatchReport` | 对每组用 `G27970{字母}` 取小组赛，取**第 5、6 场**作为最后一轮，比较两场 **开球时间是否相同**，输出同时开赛与否对默契球风险的提示。**依赖 `c75.js` 中的小组赛程结构**。 |
| `fatigue-schedule.md` | `generateFatigueReport` | 汇总各队比赛时间，计算相邻场次 **间隔天数**，标记间隔 ≤3 天的「紧密赛程」等。**跨美加墨旅行距离未在表中量化**，正文中有文字提醒。 |

另：**`strategy/README.md`** 由 `generateStrategyReadme` 生成，用表格说明上述 4 个文件的用途及赛中何时参考。

---

## 4. 与 workflow「阶段三」的关系

- **阶段一第 5 步**：运行脚本，生成/覆盖 **`strategy/`** 下 Markdown **底稿**，便于赛前统一框架。
- **阶段三（贯穿赛中）**：积分、战意、挑对手、默契球评估等需结合**实时赛果**人工更新或补充；脚本内虽有 `identifyRotationRisk` 等辅助逻辑，**当前 `generateFullReport` 未调用**，轮换与战意仍以文档 + 人工分析为主。

---

## 5. 小结

在 **`cup-analyzer/crawler-server`** 下执行 **`node processors/strategyAnalyzer.js`**（必要时 `CUP_ANALYZER_CUP=theWorldCup`）→ 读取 **`theWorldCup/data/c75.js`** 与内置淘汰赛对阵表 → 输出到 **`theWorldCup/strategy/`**：**4 份策略分析 `.md`** + **`README.md`**。

更细的字段含义与杯赛数据约定见 `workflow.md` 中「赛程数据」一节及 `crawler-server/README.md` 中 **`strategyAnalyzer.js`** 说明。
