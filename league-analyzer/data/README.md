文件的命名：s代表联赛，c代表杯赛。处理逻辑在工程的根目录的backend-server文件夹里有
英文字母后的数字代表联赛序号，如36代表英超。所以s36就是英超联赛的数据，后缀是js文件
这是这个赛季的数据
然后解释文件里的内容：
arrTeam是球队的信息，二维数组里每个元素就是一只球队的信息每个元素数组下标0代表球队序号，下标1代表中文名
jh["R_X"],X代表第X轮的赛事信息，二维数组里每个元素就是一场比赛，每个元素数组下标0代表比赛序号，下标1代表联赛序号，下标2比赛状态，下标3代表开赛时间，下标4代表主队球队序号，下标5客队球队序号，下标6代表全场比分，下标7代表半场比分，下标10代表亚盘全场初盘，下标12代表大小盘全场初盘
totalScore是联赛积分榜

l开头的是亚让盘路榜数据，TotalPanLu是总盘盘路榜，HomePanLu是主场盘路榜，GuestPanLu是客场盘路榜
这三个变量的二维数组里每个元素就是一个球队的亚盘信息，每个元素数组下标0代表排名，下标1代表球队序号，下标2已结束比赛轮数，下标6代表赢盘数，下标7代表走盘数，下标8输盘数，下标9代表净胜盘数，下标10代表亚盘胜率

bs开头的是亚让盘路榜数据，TotalBs是总大小盘路榜，HomeBs是主场大小榜，GuestBs是客场大小榜
这三个变量的二维数组里每个元素就是一个球队的大小球信息，每个元素数组下标0代表排名，下标1代表球队序号，下标2已结束比赛轮数，下标3代表大球数，下标4代表走盘数，下标5小球数，下标6代表打出大球率

td开头的是入球时间数据，arrData的二维数组里每个元素就是一个球队的入球时间信息，每个元素数组下标0代表球队序号，下标1~18代表不同时间段的进球数
时间段分布：下标1(1-5分钟)，下标2(6-10分钟)，下标3(11-15分钟)，下标4(16-20分钟)，下标5(21-25分钟)，下标6(26-30分钟)，下标7(31-35分钟)，下标8(36-40分钟)，下标9(41-45分钟)，下标10(46-50分钟)，下标11(51-55分钟)，下标12(56-60分钟)，下标13(61-65分钟)，下标14(66-70分钟)，下标15(71-75分钟)，下标16(76-80分钟)，下标17(81-85分钟)，下标18(86-90分钟)
75分钟后进球数计算方法：下标16 + 下标17 + 下标18
