# 比赛分析工作流程

**项目**: league-analyzer  
**创建时间**: 2025-11-28  
**更新时间**: 2025-11-28

---

## 工作流程总览

比赛分析报告不是一次生成就完美的，需要反复修改，逐步完善，最终确定投注方案。

整个流程分为以下步骤：

1. **搜集球队大名单** ✅ (本文档)
2. 预测主客队首发阵容
3. 查看赛前新闻和发布会
4. 分析盘口和赔率
5. 综合判断和决策
6. 观看直播Live
7. 赛后复盘

## 第一步：搜集球队大名单

### 目标
获取主队和客队的完整大名单信息，包括每个球员的出场次数、位置、进球助攻数据等。

大名单已经由用户保存在`league-analyzer/news/${联赛名}/${赛季}/${轮次}/${对阵双方}` 文件夹里了，统计信息（对战、近期战绩、未来比赛）也在里面。

#### 首发阵容格式规范 ⭐⭐⭐

**格式说明**:
- 位置排列：从右向左（门将/后卫线/中场线/前场线/前锋）
- 分隔符：用 `/` 分隔不同位置线，用 `,` 分隔同一位置线的球员
- 替补说明：在括号内标注位置
- **必须包含四个部分**：预测首发（11人）、替补（9人）、伤停、落选

**完整格式示例 - 曼城 (4-1-4-1)**:
```
预测首发：25-多纳鲁马/27-努内斯，3-迪亚斯，24-格瓦迪奥尔，33-尼科奥赖利/14-尼古拉斯.冈萨雷斯/11-多库，20-贝尔纳多·席尔瓦，4-赖因德斯，47-福登/9-哈兰德

替补：1-特拉福德(GK)，82-里科·刘易斯(RB)，5-斯通斯(CB)，45-胡桑诺夫(CB)，21-艾特努里(LB)，26-萨维尼奥(LW)，7-马尔穆什(LW)，10-切尔基(CAM)，52-奥斯卡·鲍伯(RW)

伤停：
- 16-罗德里·赫尔南德斯（29岁，191cm，西班牙，9000万，6场4首发）- 腿筋伤
- 8-马特奥·科瓦契奇（31岁，177cm，克罗地亚，2000万，1场0首发）
- 44-菲利普斯（30岁，179cm，英格兰，1000万）

落选：
- 6-内森·阿克（30岁，180cm，荷兰，2200万，5场1首发）
```

**完整格式示例 - 切尔西 (4-2-3-1)**:
```
预测首发：1-桑切斯/24-R.詹姆斯，34-阿昌蓬，23-查洛巴，3-库库雷利亚/25-凯塞多，8-恩佐/7-内托，20-若奥佩德罗，49-加纳乔/41-埃斯特旺

替补：12-乔金森(GK)，27-古斯托(RB)，29-福法纳(CB)，21-哈托(LB)，45-拉维亚(CDM)，10-帕尔默(CAM)，11-吉滕斯(LW)，9-德拉普(ST)，32-乔治(FW)

伤停：
- 6-列维·科尔威尔（22岁，187cm，英格兰，5500万）- 十字韧带拉伤，本赛季报销
- 斯特林（31岁，170cm，英格兰，700万）- 下放青年队，非一线队
- 穆德里克（24岁，175cm，乌克兰，0万）- 国际足联禁赛

落选：
- 44-加布里埃尔·斯隆纳（21岁，193cm，美国，350万）
- 4-托辛·阿达拉比奥约（28岁，197cm，英格兰，2000万，8场4首发）
- ... （其他未在预测首发、替补、伤停中的球员）
```

**格式要点**:
1. **预测首发**（11人）：严格按照阵型排列，排除伤停人员
   - 门将单独一组
   - 后卫线从右到左：右后卫，右中卫，左中卫，左后卫
   - 中场线从右到左：根据阵型排列（4-2-3-1: 双后腰；4-3-3: 三中场；4-1-4-1: 单后腰+四前腰）
   - 前场线从右到左：边锋/前腰，中锋，边锋/前腰

2. **替补**（9人）：按位置分类，用括号标注位置缩写
   - 必须包含1个门将
   - 其余8人根据实际情况选择

3. **伤停**：必须列出
   - 确定伤停的球员（含伤情说明）
   - 长期伤停的球员（如赛季报销）
   - 非伤病原因不能上场的球员（如禁赛、下放青年队等）
   - 格式：球衣号-名字（年龄，身高，国籍，身价，出场数据）- 伤情/原因

4. **落选**：必须列出
   - 定义：在大名单内，但不在预测首发、替补、伤停中的球员
   - 包括：第三门将、边缘球员、年轻球员等
   - 格式：球衣号-名字（年龄，身高，国籍，身价，出场数据）

**⚠️ 重要原则**:
- **预测首发要尽量精准**，力求与实际首发一致
- **伤停人员必须排除**，不能出现在预测首发中
- **伤疑人员**：如果是主力且可能上场，可以排进预测首发，但要在分析中说明
- **状态火热的球员优先首发** ⭐⭐⭐：
  - 即使平时不是主力，但如果近期状态火热（连续进球/助攻），应该排进预测首发
  - 示例：恩梅查虽然只有3次首发，但3场3球，状态火热，应该顶替勒温首发
  - 判断标准：
    - 连续2-3场进球/助攻
    - 替补出场效率高于主力
    - 近期表现明显优于同位置竞争者
  - 在首发分析中要特别标注状态火热的原因
- **四个部分一个都不能少**：预测首发 + 替补 + 伤停 + 落选

#### 5. 重复操作

对客队重复步骤1-3：
1. 修改配置文件为客队信息
2. 执行爬虫命令
3. 保存客队信息

#### 6. 写入比赛报告

将两队的大名单信息整理后，写入比赛分析报告的"球员信息"部分。

## 注意事项

### 1. 数据准确性
- ⭐⭐⭐ **优先查看description文件**，其中的信息优先级最高
- 爬虫数据可能有延迟，以官方最新消息为准
- 注意球员名称的多种写法（如"科尔威尔"和"列维·科尔威尔"）
- 有些球员可能未被正确识别，需要手动核对

### 2. 伤病信息
- ⭐⭐⭐ **优先查看description文件**，可能包含确切的伤病信息（如赛季报销）
- 爬虫数据中出场次数少的球员，可能有伤病
- 完全未出现的球员，需要特别确认状态
- 伤病信息需要结合赛前新闻和description文件确认
- 区分：赛季报销、长期伤停、短期伤停、疑似伤停

### 3. 阵容选择
- 根据最常用阵型推荐首发
- 选择出场次数最多且位置不重叠的球员
- 首发11人 + 替补9人 = 20人大名单

### 4. 特殊情况
- **补赛**: roundSerial 填未开赛的轮次，以获取更多数据
- **杯赛**: isNation 改为对应的杯赛代码
- **国家队**: isNation 改为 true，使用 c 开头的数据文件

## 阶段二：编写比赛分析报告 📝

### 目标
根据搜集的球队大名单信息，**严格按照固定模板**逐项完成比赛分析报告。

### 模板位置
```
league-analyzer/prompts/match_analysis_template.md
```

### ⚠️ 重要规则

**模板结构说明**:
- **汉字（一、二、三...）**= 大点/章节标题
- **阿拉伯数字（1、2、3...）**= 小点/具体分析项
- **除了第二大点"分析时必看"是警醒的话需照抄，其他所有大点的小点都要逐个完成**
- **不能自己随意发挥，不能遗漏任何小点，不能合并小点**

### 模板内容详解

#### 一、球员信息
1. **主队大名单** - 列出主队所有球员详细信息（按位置分类：门将、后卫、中场、前锋）
   - ⚠️ **格式要求**：使用"球衣号-名字"格式（如：9-哈兰德），而不是"球衣号. 名字"
   - 方便复制到其他编辑器时能看到球衣号
2. **客队大名单** - 列出客队所有球员详细信息（按位置分类）
   - ⚠️ **格式要求**：使用"球衣号-名字"格式
3. **主队伤病** - 列出主队伤停、疑似伤停、状态不明的球员，标注重要性
4. **客队伤病** - 列出客队伤停、疑似伤停、状态不明的球员，标注重要性
5. **主队预测首发** - 按首发阵容格式写出主队预测首发11人和替补9人
6. **客队预测首发** - 按首发阵容格式写出客队预测首发11人和替补9人
7. **预测总结** - 对比两队首发阵容，分析阵型、关键对位、优劣势
8. **主队实际首发** - （比赛开始前确认填写）
9. **客队实际首发** - （比赛开始前确认填写）
10. **实际首发总结** - （比赛开始前确认填写，对比预测和实际）

#### 二、分析时必看
**警醒自己的话** - 照抄模板内容（关于输的原因、避免措施、项目驱动等）

#### 三、发布会信息
1. **主队新闻** - 搜集主教练发布会、球队官方新闻、伤病更新
2. **客队新闻** - 搜集主教练发布会、球队官方新闻、伤病更新
3. **主队战意** - 分析主队本场比赛的重要性、动力、战意等级
4. **客队战意** - 分析客队本场比赛的重要性、动力、战意等级

#### 四、盘口解析
1. **亚盘** - 分析初盘、盘口含义、可能走势
   - 数据来源：澳彩(Macau)、皇冠(Crown)
   - 比赛ID获取：从`backend-server/data/s{联赛序号}.js`文件中查找
   - 访问网址：`https://vip.titan007.com/AsianOdds_n.aspx?id={比赛ID}`
   - 分析要点：
     - 广实定位：判断盘口是否符合球队档次定位
     - 受注情况：盘口升降变化
     - 拉力分析：哪个队受热（拉力强的不打出）
     - 投注平衡度：盘面是否稳定
2. **欧赔** - 分析主胜、平局、客胜赔率及变化趋势
   - 数据来源：威廉希尔(William Hill)、韦德(Bet365)
   - 访问网址：`https://1x2.titan007.com/oddslist/{比赛ID}.htm`
   - 分析要点：
     - 亚欧对应关系：欧赔是否符合亚盘对应区间
     - 欧赔结构：135/134体系分析
     - 跷跷板理论：高赔是否约为中赔的2倍
     - 综合判断：正路还是冷门
3. **大小** - 分析大小球盘口、进球预期
4. **临场赔率** - （赛前1-2小时填写）临场盘口、赔率、资金流向变化

#### 五、主客队近况
**数据来源**: 
- 用户保存在`league-analyzer/news/${联赛名}/${赛季}/${轮次}/${对阵双方}`里的文件：统计信息.md
- 包含：对战战绩、主队近期战绩、客队近期战绩、主队未来赛程、客队未来赛程

1. **对战交锋** - 历史交锋战绩、主客场记录、进球统计、特殊情况
2. **主队近况** - 近5-10场战绩、进攻表现、防守表现、关键问题
3. **客队近况** - 近5-10场战绩、进攻表现、防守表现、关键优势
4. **主队积分榜情况** - 积分、排名、主客场数据、半场数据
5. **客队积分榜情况** - 积分、排名、主客场数据、半场数据
6. **主队未来3场比赛** - 赛程分析、难度评估、体能考虑
7. **客队未来3场比赛** - 赛程分析、难度评估、体能考虑

#### 六、结果
1. **开始预测方向** - 综合分析后的初步预测（胜平负、比分、大小球、风险等级）
2. **实际下注方向** - （临场确定后填写）根据首发和临场盘口的最终下注方向
3. **星级（普通、五星、六星）** - 下注信心等级

#### 七、舆情
1. **主队舆情（冷热情况）** - 媒体预测、球迷热度、社交媒体讨论、投注分布
2. **客队舆情（冷热情况）** - 媒体预测、球迷热度、社交媒体讨论、投注分布

### 填写时机和流程

**阶段一：赛前2-3天**
- 完成"一、球员信息"（1-7小点）
- 完成"二、分析时必看"（照抄）
- 完成"五、主客队近况"（1-7小点）
- 完成"四、盘口解析"（1-3小点，初步分析）

**阶段二：赛前1天**
- 完成"三、发布会信息"（1-4小点）
- 完成"七、舆情"（1-2小点）
- 更新伤病信息

**阶段三：赛前2小时**
- 完成"一、球员信息"（8-10小点，确认实际首发）
- 完成"四、盘口解析"（第4小点，临场赔率）
- 完成"六、结果"（1-3小点，最终预测和下注方向）

**阶段四：赛后**
- 复盘分析
- 总结经验教训
- 更新到个人笔记

### ⚠️ 严格要求

1. **不能遗漏小点**: 每个小点都是分析的必要环节
2. **不能合并小点**: 每个小点都要单独成段落
3. **不能随意发挥**: 严格按照模板结构来
4. **逐步完善**: 报告不是一次写完，要分阶段填写
5. **数据支撑**: 每个分析都要有数据或事实依据

### 示例文件
```
league-analyzer/report/epl/25-26/round-13/chelsea_vs_arsenal.md
```

## 阶段三：盘口和赔率分析 📊

### 目标
分析亚盘和欧赔数据，结合球队实力、伤病、状态等因素，判断投注方向。

### 数据来源

**比赛ID获取**：
- 位置：`backend-server/data/s{联赛序号}.js`
- 示例：英超数据在`s36.js`中
- 格式：`jh["R_13"]` = 第13轮比赛数据
- 每场比赛的比赛ID在数组的第0个元素

**亚盘数据**：
- 主要公司：澳彩(Macau)、皇冠(Crown)
- 访问网址：`https://vip.titan007.com/AsianOdds_n.aspx?id={比赛ID}`
- 示例：曼城vs利兹联（ID=2789254）
  - `https://vip.titan007.com/AsianOdds_n.aspx?id=2789254`

**欧赔数据**：
- 主要公司：威廉希尔(William Hill)、韦德(Bet365)
- 访问网址：`https://1x2.titan007.com/oddslist/{比赛ID}.htm`
- 示例：曼城vs利兹联（ID=2789254）
  - `https://1x2.titan007.com/oddslist/2789254.htm`

---

### 亚盘分析方法

**⚠️⚠️⚠️ 最重要：必须主动查看网站！**

**工作原则** 🚨:
1. **不要只依赖s36.js的初盘数据！**
2. **必须使用web_search工具查看临场盘口数据！**
3. **不允许写"待查看网站"、"待补充"等字样！**
4. **完整交付，不留尾巴！**

**数据来源层级**:
1. **初盘数据**（第一层）: 
   - 来源：`data/s36.js`
   - 内容：初盘全场让球、初盘半场让球
   - ⚠️ 注意：s36.js只有初盘，没有临场盘！
   
2. **临场盘数据**（第二层，必须查看）⭐⭐⭐:
   - 来源：**必须使用web_search查看** `https://vip.titan007.com/AsianOdds_n.aspx?id={比赛ID}`
   - 内容：临场盘口、水位变化、盘口变化历史
   - ⚠️ **这一步不是可选的，是必须的！**
   
3. **欧赔数据**（第三层，必须查看）⭐⭐⭐:
   - 来源：**必须使用web_search查看** `https://1x2.titan007.com/oddslist/{比赛ID}.htm`
   - 内容：威廉希尔、Bet365等主流公司的初盘和临场盘

#### 0.5. 识别高水让盘陷阱 🚨🚨🚨

**⚠️⚠️⚠️ 极其重要：不要盲目推荐强队让盘！**

**什么是高水让盘？**
- **定义**：上盘（让球方）水位≤0.85
- **示例**：曼城让一球，水位0.80 -1 1.09
  - 曼城是上盘（让球方），水位0.80（高水）
  - 对手是下盘（受让方），水位1.09（低水）

**高水让盘的含义** 💡:
1. **庄家对让球方没有信心** - 水位低说明赔付少，庄家不看好
2. **诱盘** - 用强队名气吸引散户买让盘
3. **常见结果**：
   - 让球方小胜1球（如1-0, 2-1）→ 赢球但不赢盘 ❌
   - 被逼平 → 不赢盘 ❌
   - 爆冷输球 → 不赢盘 ❌

**识别标准** 🔍:
1. **上盘水位≤0.85** → 高水让盘
2. **临场水位进一步降低但盘口不升** → 庄家诱盘
3. **市场资金流向上盘，但庄家不升盘** → 庄家对上盘没信心

**错误示例** ❌:
```
诺丁汉森林 vs 曼城
初盘：0.81 -1 1.07（曼城高水让一球）
临场：0.80 -1 1.09（曼城水位更低）

错误分析：曼城实力强，让一球合理，推荐曼城-1
正确分析：高水让盘陷阱！曼城很可能小胜或被逼平，不推荐让盘
```

**正确做法** ✅:
1. **计算水位**：
   - 上盘水位 = 初盘数据第一个数字
   - 如果≤0.85，立即警觉
   
2. **观察临场变化**：
   - 水位降低但盘口不升 → 诱盘
   - 水位升高或盘口升 → 庄家看好上盘
   
3. **综合判断**：
   - 结合防守端数据（如曼城近5场失10球）
   - 结合对手近期状态（如森林主场3场3-0大胜）
   - 结合历史记录（如曼城客场对弱队有失利）
   
4. **投注建议**：
   - ❌ **强烈不推荐高水让盘** - 0%资金
   - ✅ 可以考虑独赢（不让球） - 30-40%资金
   - ✅ 可以考虑大小球 - 50-60%资金
   - ✅ 或干脆避开本场比赛

**为什么不能盲目推荐强队？** 💭:
1. **实力差距≠盘口优势** - 强队实力强不代表能赢盘
2. **水位是关键** - 高水说明庄家不看好
3. **历史数据很重要** - 强队客场对弱队常有意外
4. **要站在庄家角度** - 庄家为什么不愿意升盘？

**必须记住** ⚠️:
- 看到上盘水位≤0.85，立即打问号
- 高水让盘不推荐，避开或买独赢
- 不要被强队名气迷惑
- 不要盲目推荐亚盘上盘

### 欧赔分析方法

**核心理念**：
- **亚欧结合**：欧赔必须结合亚盘看
- **跷跷板理论**：以平赔为轴，高赔约为中赔的2倍时，低赔打出

**分析步骤**：

#### 1. 亚欧对应关系

**标准对应区间**：
- 亚盘平手 → 欧赔最低赔 2.3~2.6
- 亚盘平半 → 欧赔最低赔 2.1~2.3
- 亚盘半球 → 欧赔最低赔 1.8~2.1
- 亚盘半一 → 欧赔最低赔 1.6~1.7
- 亚盘一球 → 欧赔最低赔 1.5~1.6
- 亚盘一球/球半 → 欧赔最低赔 1.4~1.5
- 亚盘球半 → 欧赔最低赔 1.3~1.4
- 亚盘球半/两球 → 欧赔最低赔 1.2~1.3 ⭐
- 亚盘两球 → 欧赔最低赔 1.1~1.2
- 亚盘两球以上 → 欧赔最低赔 1.1以下

**第一步**：确认欧赔是否在对应区间内

#### 2. 欧赔区间划分

**威廉希尔标准赔率结构**：

**一区间（平手）**：
- 2.40 3.25 2.90
- 2.50 3.20 2.87
- 2.70 3.10 2.70

**二区间（平手/半球）**：
- 2.10 3.30 3.50
- 2.20 3.30 3.40
- 2.37 3.25 3.00

**三区间（半球）**：
- 1.85 3.50 4.20
- 2.00 3.40 3.75

**四区间（半球/一球）**：
- 1.70 3.80 4.80
- 1.80 3.60 4.40

**五区间（一球）**：
- 1.50 4.20 6.50
- 1.60 3.80 5.50

**六区间及以上（球半以上）**：
- 1.25 5.80 11.00（球半/两球）⭐

#### 3. 跷跷板理论

**基本原理**：
- 以平赔为轴，资金在胜赔和负赔之间流动
- 当高赔是中赔的约2倍时，结构稳定，低赔打出

**判断标准**：
- ✅ 高赔/中赔 ≈ 2倍 → 正路赛果，低赔打出
- ⚠️ 高赔/中赔 > 2倍 → 高赔虚高，低赔打出可能性大
- ⚠️ 高赔/中赔 < 2倍 → 中高赔偏低，要防冷门

**特殊情况**：
- 最低赔 ≥ 2.0：跷跷板理论完全适用
- 1.5 ≤ 最低赔 < 2.0（半球、半一）：
  - 135体系正常（如1.7 3.x 5.x）→ 主胜打出
  - 134体系要防冷门（如1.7 3.x 4.x）→ 负赔高开
- 最低赔 < 1.5（一球以上）：
  - 负赔比平赔接近2倍或超过2倍 → 主胜没问题
  - 中赔和高赔开太低 → 要防冷门

#### 4. 综合判断

结合以下因素：
- 亚欧对应（是否符合）
- 欧赔区间（哪个区间）
- 跷跷板理论（比例关系）
- 135/134体系（结构分析）
- 投注建议（正路或冷门）

- 🔥 **大球率**：
  - 60%以上：明显走大球
  - 40%-60%：中性
  - 40%以下：明显走小球
- 📊 **排名**：排名越靠前大球率越高
- 🏠 **主客场差异**：注意主客场大球率差异
- ⚠️ **矛盾点**：
  - 主队走大球 + 客队走大球 → 大球无疑
  - 主队走小球 + 客队走小球 → 小球无疑
  - 主队走大球 + 客队走小球 → 分析主队进攻还是客队防守占主导
  - 主队走小球 + 客队走大球 → 分析客队防守差还是主队进攻弱

#### 2. 分析球队进攻和防守

**进攻端**：
- 场均进球数
- 主力前锋状态
- 进攻阵容完整性
- 近期进球趋势

**防守端**：
- 场均失球数
- 防守阵容完整性
- 后防线伤病情况
- 近期失球趋势

#### 3. 75分钟后进球数分析 🕒

**核心价值**：
- 了解球队在比赛末段（76-90分钟）的进球能力
- 判断进球时间分布，是否集中在前中段还是末段
- 对大小球和比分预测提供时间维度参考

**数据来源**：
- 文件位置：`backend-server/data/td{联赛序号}.js`
- 示例：英超数据在`td36.js`中
- 数据结构：`arrData`

**数据说明**：
- 每个元素数组：`[球队序号, 1~5分钟进球, ..., 76~80分钟进球, 81~85分钟进球, 86~90分钟进球]`
- 下标0：球队序号
- 下标16：76~80分钟进球数
- 下标17：81~85分钟进球数
- 下标18：86~90分钟进球数

**分析步骤**：

**第一步：计算所有球队75分钟后进球数**
```python
# 75分钟后进球数 = 下标16 + 下标17 + 下标18
total_goals_after_75 = team_data[16] + team_data[17] + team_data[18]
```

**第二步：按进球数从大到小排序**
- 展示前10名球队
- 找到本场比赛双方球队的数据和排名

**展示格式**：
```
英超75分钟后进球数排名（TOP 10）:
1. 利物浦：8球 (76-80: 1球, 81-85: 1球, 86-90: 6球)
2. 布莱顿：8球 (76-80: 1球, 81-85: 2球, 86-90: 5球)
...

本场比赛球队数据:
- 曼彻斯特城：3球，排第14 (76-80: 0球, 81-85: 1球, 86-90: 2球)
- 利兹联：1球，排第20 (76-80: 0球, 81-85: 1球, 86-90: 0球)
```

**分析要点**：

**高进球数球队（6球以上）**：
- 说明末段进攻火力强或经常落后需要追分
- 比赛可能有悬念，末段有进球爆发

**低进球数球队（3球以下）**：
- 说明末段进球能力弱或通常前期就已取得优势
- 如果是强队，可能前期就已领先，末段控节奏
- 如果是弱队，说明末段进攻乏力，难以追分

**对大小球的影响**：
- 双方75分钟后进球数都高 → 末段可能有进球，支持大球
- 双方75分钟后进球数都低 → 进球集中在前中段
- 强队进球数低+弱队进球数低 → 如果强队大比分领先，末段可能控节奏
- 强队进球数高 → 强队可能末段继续进攻，支持大球

**综合判断示例**：
- 曼城75分钟后进球少（3球，第14）→ 通常前期就已取得优势
- 利兹联75分钟后进球极少（1球，第20）→ 末段进攻乏力
- **结论**：进球可能集中在前75分钟，但如果曼城要刷净胜球，末段仍可能进球

---

#### 4. 综合判断

结合以下因素：
- 大小球盘路榜（双方特征）
- 进攻和防守数据
- 75分钟后进球数（时间分布）⭐ 新增
- 球队状态和伤病
- 比赛性质和战意
- 投注建议（大球或小球）

**特殊情况处理**：
- 当双方盘路特征对冲时（一方走大球，一方走小球）：
  - 看防守差的一方大球率是否更高 → 大球
  - 看进攻强的一方是否主场作战 → 大球
  - 看实力差距，强队可能狂攻 → 大球
  - 综合判断谁的特征占主导

### 近况和赛程分析方法

**⚠️⚠️⚠️ 严重错误警告：客队近期战绩的数据处理** 🚨

**常见错误** ❌:
```
错误示例：伯恩茅斯最近6场战绩
1. 第12轮：主场2-2战平伯恩茅斯  ← 荒谬！球队不能和自己比赛
2. 第9轮：客场2-3战胜狼队  ← 错误！2-3是输球不是战胜
```

**错误原因**:
- 在处理客队数据时，没有正确识别对阵球队的序号
- s36.js格式：`[比赛ID, ..., 主队ID, 客队ID, ...]`
- 当球队A是客队时，对手应该是**主队ID**，不是客队ID！

**正确处理逻辑** ✅:
```
for (该球队的每场比赛) {
  if (match.主队ID == 该球队ID) {
    // 该球队是主队
    对手 = match.客队ID  ← 从客队序号获取
    主客 = "主场"
  } else if (match.客队ID == 该球队ID) {
    // 该球队是客队
    对手 = match.主队ID  ← 从主队序号获取！这里容易错！
    主客 = "客场"
  }
}
```

**必须自我检查** 🔍:
1. ✅ 生成数据后，逐条检查是否合理
2. ✅ 看到"球队A vs 球队A"立即修正
3. ✅ 检查比分和结果是否匹配（"2-3战胜"明显错误）
4. ✅ 检查主客场标注是否正确

---

**核心价值**：
- 了解球队近期状态（进攻、防守、士气）
- 分析历史交锋记录，找出对阵规律
- 评估未来赛程密度和体能分配
- 判断本场比赛的战意和重要性

**数据来源**：

**综合分析页面** ⭐ 最推荐：
- 访问网址：`https://zq.titan007.com/analysis/{比赛ID}cn.htm`
- 示例：曼城vs利兹联（ID=2789254）
  - `https://zq.titan007.com/analysis/2789254cn.htm`

**页面包含内容**：
1. **对战战绩**：历史交锋记录、主客场成绩、进球统计
2. **主队近期战绩**：最近5-10场比赛结果、进球失球
3. **客队近期战绩**：最近5-10场比赛结果、进球失球
4. **主队未来赛程**：未来3-5场比赛对手、难度评估
5. **客队未来赛程**：未来3-5场比赛对手、难度评估

---

#### 1. 对战交锋分析

**分析要点**：
- 📊 **历史战绩**：胜平负次数、主客场表现
- ⚽ **进球统计**：平均进球数、大比分场次
- 🎯 **对阵特点**：是否有克星关系、特殊规律
- 📅 **时间跨度**：近几年的交锋记录更有参考价值

**关键指标**：
- 主队主场对客队的胜率
- 客队客场对主队的胜率
- 历史交锋的大小球趋势
- 是否有明显的心理优势

---

#### 2. 近期战绩分析

**主队近况要点**：
- 🏆 **战绩**：近5-10场的胜平负情况
- ⚽ **进攻**：场均进球、进攻效率、关键球员状态
- 🛡️ **防守**：场均失球、防守稳定性、后防伤病
- 📈 **趋势**：上升期、下降期、稳定期
- 💪 **士气**：连胜/连败对士气的影响

**客队近况要点**：
- 同上，但要特别关注客场表现
- 客队客场战绩可能与总体战绩有较大差异

**对比分析**：
- 进攻对比：谁的进攻更强
- 防守对比：谁的防守更稳
- 状态对比：谁处于上升期/下降期
- 士气对比：谁的士气更高

---

#### 3. 积分榜情况分析

**分析要点**：
- 📊 **排名**：积分榜位置反映整体实力
- 🏠 **主客场差异**：主场龙/客场虫现象
- ⏰ **半场数据**：上半场/下半场表现差异
- 🎯 **保级/争冠**：积分榜位置决定战意

**关键指标**：
- 主队主场积分、排名
- 客队客场积分、排名
- 与理论排名的偏差
- 近期积分走势

---

#### 4. 未来赛程分析

**核心作用**：
- 判断球队是否会有所保留
- 评估体能分配策略
- 预测轮换可能性

**主队未来赛程要点**：
- 🗓️ **赛程密度**：多久一场，是否连续作战
- 💪 **对手难度**：未来对手强弱、重要性
- 🎯 **战略重要性**：本场是否会为未来保留实力
- ⚽ **杯赛干扰**：是否有杯赛任务

**客队未来赛程要点**：
- 同上，特别关注客场赛程安排

**综合判断**：
- 💡 **本场战意对比**：
  - 主队未来轻松 + 客队未来艰难 → 主队可能全力以赴
  - 主队未来艰难 + 客队未来轻松 → 主队可能有所保留
  - 双方都轻松 → 看积分榜位置和保级/争冠压力
  - 双方都艰难 → 都会全力以赴

- 🔄 **轮换预测**：
  - 赛程密集 + 未来有强敌 → 可能轮换主力
  - 赛程宽松 + 保级/争冠压力大 → 全主力出战

---

#### 5. 综合战意评估

结合以下因素：
- 积分榜位置（保级、争冠、欧战）
- 近期状态（连胜/连败）
- 历史交锋（心理优势）
- 未来赛程（体能分配）
- 杯赛任务（精力分散）

**战意等级**：
- ⭐⭐⭐⭐⭐ 极强：保级关键战、争冠决战
- ⭐⭐⭐⭐ 强：重要的积分争夺战
- ⭐⭐⭐ 中等：正常联赛场次
- ⭐⭐ 一般：无欲无求的中游对决
- ⭐ 弱：已经保级/无缘争冠的鸡肋战
