## 工作流程总览

对于一场比赛赛前预测，每场比赛有不同的情况，需要从基础数据中挖掘有用的信息，以了解比赛的看点。
预测相比赛后分析要相对难一些，因为你需要有更多的数据作支撑。
比赛分析报告不是一次生成就完美的，需要反复修改，逐步完善，最终确定投注方案。

整个流程分为以下步骤：

1. **搜集对阵双方的球队大名单** 
2. **预测主客队首发阵容**
3. **查看双方的交锋历史、主队近况、客队近况、积分榜情况、未来比赛**
4. **分析盘口和赔率**
5. **综合判断和决策**

### ⭐⭐⭐ 投注决策权重排序（核心原则）

**最终决定买哪个盘口，必须按以下权重排序**：

| 优先级 | 决策依据 | 权重 | 说明 |
|--------|---------|------|------|
| 🥇 第一 | **广实定位** | 最高 | 盘口是否符合球队档次定位，偏离说明什么 |
| 🥇 第一 | **亚盘盘路榜** | 最高 | 双方净胜盘数、主客场盘路排名 |
| 🥇 第一 | **大小盘路榜** | 最高 | 双方大球率、主客场大小球排名 |
| 🥈 第二 | **水位/拉力分析** | 高 | 高水让盘识别、受注方向、诱盘判断 |
| 🥈 第二 | **伤停和首发** | 高 | 关键球员缺阵对实力的影响 |
| 🥉 第三 | **历史交锋/近况/未来赛程** | 参考 | 只作背景参考，不作为决策主依据 |

**⚠️⚠️⚠️ 为什么第三层不能作为决策主依据？**

历史交锋、球队近况、未来赛程都有**反面解读**的可能：

| 常见思维误区 | 反面解读 |
|-------------|---------|
| 「历史交锋差，这队打不过对方」 | 历史交锋差 → 本场可能复仇，反弹动力更强 |
| 「近期状态差，这队不值得买」 | 近况差 → 本场可能触底反弹，否极泰来 |
| 「未来赛程凶险，会保留实力」 | 未来赛程凶险 → 可能从本场开始全力拼，调出最好状态 |
| 「连胜中，状态好继续赢」 | 连胜中 → 可能到了回调节点，盛极而衰 |

**结论**：这些因素可以正面解读，也可以反面解读，具有两面性，不具备确定性。而广实定位和盘路榜是客观数据，与投注结果的相关性最强，必须作为决策核心。

## 第一步：搜集球队大名单

### 目标
获取主队和客队的完整大名单信息，包括每个球员的出场次数、位置、进球助攻数据等。

大名单已经由用户保存在`league-analyzer/news/${联赛名}/${赛季}/${轮次}/${对阵双方}` 文件夹里了，统计信息（对战、近期战绩、未来比赛）也在里面。

#### 首发阵容格式规范 ⭐⭐⭐

**格式说明**:
- 位置排列：从右向左（门将/后卫线/中场线/前场线/前锋）
- 分隔符：用 `/` 分隔不同位置线，用 `,` 分隔同一位置线的球员
- 替补说明：在括号内标注位置
- **必须包含四个部分**：预测首发（11人）、替补（9人）、伤停、落选

**完整格式示例 - 曼城 (4-1-4-1)**:
```
预测首发：25-多纳鲁马/27-努内斯，3-迪亚斯，24-格瓦迪奥尔，33-尼科奥赖利/14-尼古拉斯.冈萨雷斯/11-多库，20-贝尔纳多·席尔瓦，4-赖因德斯，47-福登/9-哈兰德

替补：1-特拉福德(GK)，82-里科·刘易斯(RB)，5-斯通斯(CB)，45-胡桑诺夫(CB)，21-艾特努里(LB)，26-萨维尼奥(LW)，7-马尔穆什(LW)，10-切尔基(CAM)，52-奥斯卡·鲍伯(RW)

伤停：
- 16-罗德里·赫尔南德斯（29岁，191cm，西班牙，9000万，6场4首发）- 腿筋伤
- 8-马特奥·科瓦契奇（31岁，177cm，克罗地亚，2000万，1场0首发）
- 44-菲利普斯（30岁，179cm，英格兰，1000万）

落选：
- 6-内森·阿克（30岁，180cm，荷兰，2200万，5场1首发）
```

**完整格式示例 - 切尔西 (4-2-3-1)**:
```
预测首发：1-桑切斯/24-R.詹姆斯，34-阿昌蓬，23-查洛巴，3-库库雷利亚/25-凯塞多，8-恩佐/7-内托，20-若奥佩德罗，49-加纳乔/41-埃斯特旺

替补：12-乔金森(GK)，27-古斯托(RB)，29-福法纳(CB)，21-哈托(LB)，45-拉维亚(CDM)，10-帕尔默(CAM)，11-吉滕斯(LW)，9-德拉普(ST)，32-乔治(FW)

伤停：
- 6-列维·科尔威尔（22岁，187cm，英格兰，5500万）- 十字韧带拉伤，本赛季报销
- 斯特林（31岁，170cm，英格兰，700万）- 下放青年队，非一线队
- 穆德里克（24岁，175cm，乌克兰，0万）- 国际足联禁赛

落选：
- 44-加布里埃尔·斯隆纳（21岁，193cm，美国，350万）
- 4-托辛·阿达拉比奥约（28岁，197cm，英格兰，2000万，8场4首发）
- ... （其他未在预测首发、替补、伤停中的球员）
```

**格式要点**:
1. **预测首发**（11人）：严格按照阵型排列，排除伤停人员
   - 门将单独一组
   - 后卫线从右到左：右后卫，右中卫，左中卫，左后卫
   - 中场线从右到左：根据阵型排列（4-2-3-1: 双后腰；4-3-3: 三中场；4-1-4-1: 单后腰+四前腰）
   - 前场线从右到左：边锋/前腰，中锋，边锋/前腰

2. **替补**（9人）：按位置分类，用括号标注位置缩写
   - 必须包含1个门将
   - 其余8人根据实际情况选择

3. **伤停**：必须列出
   - 确定伤停的球员（含伤情说明）
   - 长期伤停的球员（如赛季报销）
   - 非伤病原因不能上场的球员（如禁赛、下放青年队等）
   - 格式：球衣号-名字（年龄，身高，国籍，身价，出场数据）- 伤情/原因

4. **落选**：必须列出
   - 定义：在大名单内，但不在预测首发、替补、伤停中的球员
   - 包括：第三门将、边缘球员、年轻球员等
   - 格式：球衣号-名字（年龄，身高，国籍，身价，出场数据）

**⚠️ 重要原则**:
- **预测首发要尽量精准**，力求与实际首发一致
- **伤停人员必须排除**，不能出现在预测首发中
- **伤疑人员**：如果是主力且可能上场，可以排进预测首发，但要在分析中说明
- **状态火热的球员优先首发** ⭐⭐⭐：
  - 即使平时不是主力，但如果近期状态火热（连续进球/助攻），应该排进预测首发
  - 示例：恩梅查虽然只有3次首发，但3场3球，状态火热，应该顶替勒温首发
  - 判断标准：
    - 连续2-3场进球/助攻
    - 替补出场效率高于主力
    - 近期表现明显优于同位置竞争者
  - 在首发分析中要特别标注状态火热的原因
- **四个部分一个都不能少**：预测首发 + 替补 + 伤停 + 落选

#### 6. 写入比赛报告

将两队的大名单信息整理后，写入比赛分析报告的"球员信息"部分。

## 注意事项

### 1. 数据准确性
- 爬虫数据可能有延迟，以官方最新消息为准
- 注意球员名称的多种写法（如"科尔威尔"和"列维·科尔威尔"）
- 有些球员可能未被正确识别，需要手动核对

### 2. 伤病信息
- 爬虫数据中出场次数少的球员，可能有伤病、可能是新援
- 完全未出现的球员，需要特别确认状态
- 伤病信息需要结合赛前新闻和description文件确认
- 区分：赛季报销、长期伤停、短期伤停、疑似伤停

### 3. 阵容选择
- 根据最常用阵型推荐首发
- 选择出场次数最多且位置不重叠的球员
- 首发11人 + 替补9人 = 20人大名单

### 4. 特殊情况
- **补赛**: roundSerial 填未开赛的轮次，以获取更多数据
- **杯赛**: isNation 改为对应的杯赛代码
- **国家队**: isNation 改为 true，使用 c 开头的数据文件

## 阶段二：编写比赛分析报告 📝

### 🚨 历史同赔分析 - 每次必执行（不可跳过）

**每次**生成赛前分析报告时，**必须**在撰写「四、盘口解析」第3小点（历史同赔分析）**之前**执行以下步骤：

1. **配置**：在 `backend-server/config/wudaconfig.js` 中设置 `matchSerial`（比赛ID）和 `roundSerial`（轮次，如18）为当前比赛
2. **执行**：在 `backend-server` 目录下运行 `node crawlerHistoryHandicap.js`
3. **写入**：将脚本输出的 Part 1（历史同赔走势）和 Part 2（同档次对阵分析）完整整理到报告中

**禁止行为**：
- ❌ 禁止跳过运行脚本
- ❌ 禁止写「待补充」「待运行」「无数据」等占位而不实际执行
- ❌ 禁止编造或猜测同赔数据

**无论报告是新建还是更新，只要涉及盘口解析，都必须运行该脚本。**

### 目标
根据搜集的球队大名单信息，**严格按照固定模板**逐项完成比赛分析报告。

### 模板位置
```
league-analyzer/prompts/match_analysis_template.md
```

### ⚠️ 重要规则

**模板结构说明**:
- **汉字（一、二、三...）**= 大点/章节标题
- **阿拉伯数字（1、2、3...）**= 小点/具体分析项
- **除了第二大点"分析时必看"是警醒的话需照抄，其他所有大点的小点都要逐个完成**
- **不能自己随意发挥，不能遗漏任何小点，不能合并小点**

### 模板内容详解

#### 一、球员信息
1. **主队大名单** - 列出主队所有球员详细信息（按位置分类：门将、后卫、中场、前锋）
   - ⚠️ **格式要求**：使用"球衣号-名字"格式（如：9-哈兰德），而不是"球衣号. 名字"
   - 方便复制到其他编辑器时能看到球衣号
2. **客队大名单** - 列出客队所有球员详细信息（按位置分类）
   - ⚠️ **格式要求**：使用"球衣号-名字"格式
3. **主队伤病** - 列出主队伤停、疑似伤停、状态不明的球员，标注重要性
4. **客队伤病** - 列出客队伤停、疑似伤停、状态不明的球员，标注重要性
5. **主队预测首发** - 按首发阵容格式写出主队预测首发11人和替补9人
6. **客队预测首发** - 按首发阵容格式写出客队预测首发11人和替补9人
7. **预测总结** - 对比两队首发阵容，分析阵型、关键对位、优劣势
8. **主队实际首发** - （比赛开始前确认填写）**必须**包含表格：位置、球衣号、球员名、数据、备注
9. **客队实际首发** - （比赛开始前确认填写）**必须**包含表格：位置、球衣号、球员名、数据、备注
10. **实际首发总结** - （比赛开始前确认填写，对比预测和实际）

**⚠️ 实际首发表格格式（8、9小点必执行）**：
- 主队、客队实际首发**必须**以表格形式列出每位首发球员的：**位置、球衣号、球员名、数据、备注**
- 表格格式示例：
| 位置 | 球衣号 | 球员名 | 数据 | 备注 |
|------|--------|--------|------|------|
| GK | 12 | 米雷斯 | 17场17首发 | 主力门将 |
| RB | 23 | 格兰特 | 17场17首发，1助 | 主力右后卫 |
| ... | ... | ... | ... | ... |
- 数据：出场次数、首发次数、进球助攻等
- 备注：主力/轮换、顶替谁、伤停复出等关键信息

#### 二、分析时必看
**警醒自己的话** - 照抄模板内容（关于输的原因、避免措施、项目驱动等）

#### 三、发布会信息
1. **主队新闻** - 搜集主教练发布会、球队官方新闻、伤病更新
2. **客队新闻** - 搜集主教练发布会、球队官方新闻、伤病更新
3. **主队战意** - 分析主队本场比赛的重要性、动力、战意等级
4. **客队战意** - 分析客队本场比赛的重要性、动力、战意等级

#### 四、盘口解析

#### 0.5. 识别高水让盘陷阱 🚨🚨🚨

**⚠️⚠️⚠️ 极其重要：不要盲目推荐强队让盘！**

**什么是高水让盘？**
- **定义**：上盘（让球方）水位≤0.85
- **示例**：曼城让一球，水位0.80 -1 1.09
  - 曼城是上盘（让球方），水位0.80（高水）
  - 对手是下盘（受让方），水位1.09（低水）

**高水让盘的含义** 💡:
1. **庄家对让球方没有信心** - 水位低说明赔付少，庄家不看好
2. **诱盘** - 用强队名气吸引散户买让盘
3. **常见结果**：
   - 让球方小胜1球（如1-0, 2-1）→ 赢球但不赢盘 ❌
   - 被逼平 → 不赢盘 ❌
   - 爆冷输球 → 不赢盘 ❌

**识别标准** 🔍:
1. **上盘水位≤0.85** → 高水让盘
2. **临场水位进一步降低但盘口不升** → 庄家诱盘
3. **市场资金流向上盘，但庄家不升盘** → 庄家对上盘没信心

**错误示例** ❌:
```
诺丁汉森林 vs 曼城
初盘：0.81 -1 1.07（曼城高水让一球）
临场：0.80 -1 1.09（曼城水位更低）

错误分析：曼城实力强，让一球合理，推荐曼城-1
正确分析：高水让盘陷阱！曼城很可能小胜或被逼平，不推荐让盘
```

**正确做法** ✅:
1. **计算水位**：
   - 上盘水位 = 初盘数据第一个数字
   - 如果≤0.85，立即警觉
   
2. **观察临场变化**：
   - 水位降低但盘口不升 → 诱盘
   - 水位升高或盘口升 → 庄家看好上盘
   
3. **综合判断**：
   - 结合防守端数据（如曼城近5场失10球）
   - 结合对手近期状态（如森林主场3场3-0大胜）
   - 结合历史记录（如曼城客场对弱队有失利）
   
4. **投注建议**：
   - ❌ **强烈不推荐高水让盘** - 0%资金
   - ✅ 可以考虑独赢（不让球） - 30-40%资金
   - ✅ 可以考虑大小球 - 50-60%资金
   - ✅ 或干脆避开本场比赛

**为什么不能盲目推荐强队？** 💭:
1. **实力差距≠盘口优势** - 强队实力强不代表能赢盘
2. **水位是关键** - 高水说明庄家不看好
3. **历史数据很重要** - 强队客场对弱队常有意外
4. **要站在庄家角度** - 庄家为什么不愿意升盘？

**必须记住** ⚠️:
- 看到上盘水位≤0.85，立即打问号
- 高水让盘不推荐，避开或买独赢
- 不要被强队名气迷惑
- 不要盲目推荐亚盘上盘

#### 1. 确定广实定位

广实定位文件在`${联赛名}-team-strength.md`

**说明**：
- 主客对调大约降三个盘
- 如果没有广实定位，以庄家初盘为参考

#### 2. 分析初盘和临盘

**关键要素**：
- 初盘：开盘时的盘口和水位
- 临盘：临场时的盘口和水位
- 盘口变化：升盘/降盘/维持
- 水位变化：升水/降水

**判断标准**：
- ✅ 盘口符合广实定位 → 合理，正常分析
- ⚠️ 盘口高于广实定位 → 庄家看好上盘（让球方），**推荐上盘**
- ⚠️ 盘口低于广实定位 → 庄家不看好上盘（让球方），**推荐下盘**
- 📊 盘面稳定 → 投注平衡
- 📈 盘口升高 → 主队受热（拉力）
- 📉 盘口降低 → 客队受热（拉力）

**⭐⭐⭐ 广实定位偏离的决策逻辑（核心原则）**：

盘口偏离的本质：庄家在纯看纸面实力（双方无伤病、状态正常）的基础上，通过降盘或升盘来表达对比赛走向的判断。

| 偏离方向 | 含义 | 推荐方向 | 示例 |
|---------|------|---------|------|
| 实际盘口 **低于** 理论盘口 | 庄家不看好上盘能赢 | **推荐下盘（受让方）** | 理论让半一(0.5/0.75)，实际只让平半(0.25)，低了2个盘 → 推荐下盘 |
| 实际盘口 **高于** 理论盘口 | 庄家看好上盘能赢 | **推荐上盘（让球方）** | 理论让半球(0.5)，实际让半一(0.5/0.75)，高了1个盘 → 推荐上盘 |
| 实际盘口 **等于** 理论盘口 | 正常开盘 | 需结合盘路榜等其他因素判断 | 理论让半球(0.5)，实际让半球(0.5) → 综合分析 |

**偏离幅度参考**：
- 低1个盘（如理论0.5/0.75，实际0.5）：轻微偏离，参考价值一般
- **低2个盘（如理论0.5/0.75，实际0.25）：明显偏离，庄家明确不看好上盘，推荐下盘**
- 低3个盘及以上：严重偏离，强烈推荐下盘

**⚠️ 易犯错误**：
- ❌ 错误思路："盘口低=上盘更容易赢盘" → 这是散户思维
- ✅ 正确思路："盘口低=庄家不看好上盘，故意降盘给上盘制造看似好打的假象" → 推荐下盘

#### 3. 拉力分析

**拉力定义**：
- 拉力强 = 受热 = 投注集中 = 庄家调整盘口 = 不易打出

**判断方法**：
- 盘口升高 → 主队拉力强 → 考虑客队
- 盘口降低 → 客队拉力强 → 考虑主队
- 盘口稳定 → 无明显拉力 → 按盘口正常分析

#### 4. 综合判断（亚盘决策）

**⭐⭐⭐ 决策优先级（从高到低）**：

1. **广实定位**（最高权重）：盘口是否符合球队档次定位，偏离方向说明庄家意图
2. **亚盘盘路榜**（最高权重）：双方净胜盘数和排名，客观反映球队打盘能力
3. **水位/拉力分析**（高权重）：受注情况、盘口升降、高水让盘识别
4. **伤停和首发**（高权重）：关键球员缺阵对实力的直接影响
5. **历史交锋/近况/未来赛程**（仅参考）：有两面性，不作为决策主依据

**投注建议**（正路或冷门）：
- 当广实定位、盘路榜、水位方向一致时 → 信心较高
- 当三者出现矛盾时 → 谨慎或回避

### 大小球分析方法

**核心理念**：
- **盘路榜优先**：先看双方的大小球盘路特征
- **球队特点**：结合进攻和防守数据
- **矛盾判断**：当双方盘路特征对冲时，分析谁占主导

**分析步骤**：

#### 1. 查看大小球盘路榜 📊

**数据来源**：
- 文件位置：`backend-server/data/bs{联赛序号}.js`
- 示例：英超数据在`bs36.js`中
- 数据结构：
  - `TotalBs`：总大小盘路榜
  - `HomeBs`：主场大小榜
  - `GuestBs`：客场大小榜

**数据说明**（以TotalBs为例）：
- 每个元素数组：`[排名, 球队序号, 已结束比赛轮数, 大球数, 走盘数, 小球数, 大球率, ..., ...]`
- 下标6：大球率（关键指标，百分比）
- 下标0：排名

**展示格式** ⚠️⚠️⚠️（**严格执行，不得修改**）：
```
总盘：主队大球率XX%，排第X，客队大球率XX%，排第X
主场：主队大球率XX%，排第X，客队大球率XX%，排第X
客场：主队大球率XX%，排第X，客队大球率XX%，排第X
```

**示例**：
```
总盘：曼城大球率33.3%，排第18，利兹联大球率66.7%，排第2
主场：曼城大球率33.3%，排第12，利兹联大球率66.7%，排第5
客场：曼城大球率33.3%，排第14，利兹联大球率66.7%，排第5
```

**格式要求** 🚨：
- ✅ 一行写一个类型（总盘/主场/客场）
- ✅ 主队和客队数据写在同一行，用逗号分隔
- ✅ 格式：`类型：主队大球率XX%，排第X，客队大球率XX%，排第X`
- ❌ **不要添加额外的表情符号**
- ❌ **不要分段分点展示**
- ❌ **不要自作主张修改格式**
- ❌ **不要添加额外的说明文字**

**分析要点**：
- 🔥 **大球率**：
  - 60%以上：明显走大球
  - 40%-60%：中性
  - 40%以下：明显走小球
- 📊 **排名**：排名越靠前大球率越高
- 🏠 **主客场差异**：注意主客场大球率差异
- ⚠️ **矛盾点**：
  - 主队走大球 + 客队走大球 → 大球无疑
  - 主队走小球 + 客队走小球 → 小球无疑
  - 主队走大球 + 客队走小球 → 分析主队进攻还是客队防守占主导
  - 主队走小球 + 客队走大球 → 分析客队防守差还是主队进攻弱

#### 2. 分析球队进攻和防守

**进攻端**：
- 场均进球数
- 主力前锋状态
- 进攻阵容完整性
- 近期进球趋势

**防守端**：
- 场均失球数
- 防守阵容完整性
- 后防线伤病情况
- 近期失球趋势

#### 3. 75分钟后进球数分析 🕒

**核心价值**：
- 了解球队在比赛末段（76-90分钟）的进球能力
- 判断进球时间分布，是否集中在前中段还是末段
- 对大小球和比分预测提供时间维度参考

**数据来源**：
- 文件位置：`backend-server/data/td{联赛序号}.js`
- 示例：英超数据在`td36.js`中
- 数据结构：`arrData`

**数据说明**：
- 每个元素数组：`[球队序号, 1~5分钟进球, ..., 76~80分钟进球, 81~85分钟进球, 86~90分钟进球]`
- 下标0：球队序号
- 下标16：76~80分钟进球数
- 下标17：81~85分钟进球数
- 下标18：86~90分钟进球数

**分析步骤**：

**第一步：计算所有球队75分钟后进球数**
```python
# 75分钟后进球数 = 下标16 + 下标17 + 下标18
total_goals_after_75 = team_data[16] + team_data[17] + team_data[18]
```

**第二步：按进球数从大到小排序**
- 展示前10名球队
- 找到本场比赛双方球队的数据和排名

**展示格式**：
```
英超75分钟后进球数排名（TOP 10）:
1. 利物浦：8球 (76-80: 1球, 81-85: 1球, 86-90: 6球)
2. 布莱顿：8球 (76-80: 1球, 81-85: 2球, 86-90: 5球)
...

本场比赛球队数据:
- 曼彻斯特城：3球，排第14 (76-80: 0球, 81-85: 1球, 86-90: 2球)
- 利兹联：1球，排第20 (76-80: 0球, 81-85: 1球, 86-90: 0球)
```

**分析要点**：

**高进球数球队（6球以上）**：
- 说明末段进攻火力强或经常落后需要追分
- 比赛可能有悬念，末段有进球爆发

**低进球数球队（3球以下）**：
- 说明末段进球能力弱或通常前期就已取得优势
- 如果是强队，可能前期就已领先，末段控节奏
- 如果是弱队，说明末段进攻乏力，难以追分

**对大小球的影响**：
- 双方75分钟后进球数都高 → 末段可能有进球，支持大球
- 双方75分钟后进球数都低 → 进球集中在前中段
- 强队进球数低+弱队进球数低 → 如果强队大比分领先，末段可能控节奏
- 强队进球数高 → 强队可能末段继续进攻，支持大球

**综合判断示例**：
- 曼城75分钟后进球少（3球，第14）→ 通常前期就已取得优势
- 利兹联75分钟后进球极少（1球，第20）→ 末段进攻乏力
- **结论**：进球可能集中在前75分钟，但如果曼城要刷净胜球，末段仍可能进球

---

#### 4. 综合判断（大小球决策）

**⭐⭐⭐ 决策优先级（从高到低）**：

1. **大小球盘路榜**（最高权重）：双方大球率和排名，客观反映球队大小球特征
2. **进攻和防守数据**（高权重）：场均进球、场均失球、关键球员是否缺阵
3. **75分钟后进球数**（中权重）：时间分布特征
4. **球队近况和战意**（仅参考）：有两面性，不作为决策主依据

**特殊情况处理**：
- 当双方盘路特征对冲时（一方走大球，一方走小球）：
  - 看防守差的一方大球率是否更高 → 大球
  - 看进攻强的一方是否主场作战 → 大球
  - 看实力差距，强队可能狂攻 → 大球
  - 综合判断谁的特征占主导

### 近况和赛程分析方法

数据已记录在`league-analyzer/news/${联赛名}/${赛季}/${轮次}/${对阵双方}/统计信息.md`里。

---

#### 1. 对战交锋分析

**⚠️ 权重提醒：历史交锋仅作背景参考，不作为盘口决策的主要依据。**
- 交锋差不代表本场会输，反而可能激发复仇欲望
- 交锋好不代表本场能赢，对手可能针对性调整

**分析要点**：
- 📊 **历史战绩**：胜平负次数、主客场表现
- ⚽ **进球统计**：平均进球数、大比分场次
- 🎯 **对阵特点**：是否有克星关系、特殊规律
- 📅 **时间跨度**：近几年的交锋记录更有参考价值

**关键指标**：
- 主队主场对客队的胜率
- 客队客场对主队的胜率
- 历史交锋的大小球趋势
- 是否有明显的心理优势

---

#### 2. 近期战绩分析

**⚠️ 权重提醒：近期战绩仅作背景参考，不作为盘口决策的主要依据。**
- 近况差 → 可能触底反弹；近况好 → 可能盛极而衰
- 连败中的球队可能憋着一口气，连胜中的球队可能松懈

**主队近况要点**：
- 🏆 **战绩**：近5-10场的胜平负情况
- ⚽ **进攻**：场均进球、进攻效率、关键球员状态
- 🛡️ **防守**：场均失球、防守稳定性、后防伤病
- 📈 **趋势**：上升期、下降期、稳定期
- 💪 **士气**：连胜/连败对士气的影响

**客队近况要点**：
- 同上，但要特别关注客场表现
- 客队客场战绩可能与总体战绩有较大差异

**对比分析**：
- 进攻对比：谁的进攻更强
- 防守对比：谁的防守更稳
- 状态对比：谁处于上升期/下降期
- 士气对比：谁的士气更高

---

#### 3. 积分榜情况分析

**分析要点**：
- 📊 **排名**：积分榜位置反映整体实力
- 🏠 **主客场差异**：主场龙/客场虫现象
- ⏰ **半场数据**：上半场/下半场表现差异
- 🎯 **保级/争冠**：积分榜位置决定战意

**关键指标**：
- 主队主场积分、排名
- 客队客场积分、排名
- 与理论排名的偏差
- 近期积分走势

---

#### 4. 未来赛程分析

**⚠️ 权重提醒：未来赛程仅作背景参考，不作为盘口决策的主要依据。**
- 未来赛程凶险 → 不代表本场保留，反而可能从本场开始全力拼、调出最好状态
- 未来赛程轻松 → 不代表本场全力，可能反而松懈
- 赛程判断存在大量主观推测，不确定性极高

**核心作用**：
- 判断球队是否会有所保留（但结论不确定）
- 评估体能分配策略
- 预测轮换可能性（主要用于首发预测，而非盘口决策）

**主队未来赛程要点**：
- 🗓️ **赛程密度**：多久一场，是否连续作战
- 💪 **对手难度**：未来对手强弱、重要性
- 🎯 **战略重要性**：本场是否会为未来保留实力
- ⚽ **杯赛干扰**：是否有杯赛任务

**客队未来赛程要点**：
- 同上，特别关注客场赛程安排

**综合判断**：
- 💡 **本场战意对比**：
  - 主队未来轻松 + 客队未来艰难 → 主队可能全力以赴
  - 主队未来艰难 + 客队未来轻松 → 主队可能有所保留
  - 双方都轻松 → 看积分榜位置和保级/争冠压力
  - 双方都艰难 → 都会全力以赴

- 🔄 **轮换预测**：
  - 赛程密集 + 未来有强敌 → 可能轮换主力
  - 赛程宽松 + 保级/争冠压力大 → 全主力出战

---

#### 5. 综合战意评估

结合以下因素：
- 积分榜位置（保级、争冠、欧战）
- 近期状态（连胜/连败）
- 历史交锋（心理优势）
- 未来赛程（体能分配）
- 杯赛任务（精力分散）

**战意等级**：
- ⭐⭐⭐⭐⭐ 极强：保级关键战、争冠决战
- ⭐⭐⭐⭐ 强：重要的积分争夺战
- ⭐⭐⭐ 中等：正常联赛场次
- ⭐⭐ 一般：无欲无求的中游对决
- ⭐ 弱：已经保级/无缘争冠的鸡肋战

**⚠️ 再次提醒**：战意评估只是背景信息，写入报告作为分析参考即可。**最终盘口决策必须以广实定位、亚盘盘路榜、大小盘路榜为主要依据**，不要因为「历史交锋差」「近况差」「未来赛程凶险」就直接否定一支球队——这些因素都有反面解读的空间。
